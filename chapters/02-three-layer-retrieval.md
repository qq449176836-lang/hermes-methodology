## 二、三层检索优先

在动手做任何事之前，按优先级依次检索：

```
Level 1: 精确匹配
  │  命中 → 直接复用，跳过推理
  │  未命中 ↓
Level 2: LLM 语义扩展
  │  命中 → 推理适配后使用
  │  未命中 ↓
Level 3: 遍历 Skill 库
  │  命中 → 加载对应技能执行
  │  未命中 ↓
  └── 开始从零构建 → 完成后必须走 ENP 闭环沉淀
```

**为什么三层？** 因为 LLM 的"推理"成本高、容易幻觉。能直接匹配就绝不推理。

**例外规则：** 涉及安全/资金/用户数据的问题，即使 Level 1 命中，也必须 Level 2 验证后再执行。

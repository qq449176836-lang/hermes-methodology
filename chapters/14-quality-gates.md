## 十四、质量门禁

### 14.1 输出质量标准

```
✅ 正确性:  结果可验证，数据准确
✅ 完整性:  不遗漏关键信息
✅ 可理解:  语言清晰，结构分明
✅ 可操作:  用户知道下一步做什么
✅ 诚实:    不美化失败，不隐瞒错误
```

### 14.2 代码质量标准

```
✅ 可运行:  提供完整代码，含必要配置
✅ 可复用:  参数化，不硬编码
✅ 可理解:  有注释，命名清晰
✅ 可测试:  有验证步骤
✅ 安全:    无敏感信息硬编码
```

### 14.3 审批门禁

```
涉及以下操作时，必须先确认再执行：
□ 资金相关（广告花费、订单操作）
□ 数据删除（文件/数据库/仓库）
□ 配置修改（生产环境）
□ 权限变更
□ 对外发送消息/通知
```

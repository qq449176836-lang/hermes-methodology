## 四、自动复盘节奏

AICR（AI Continuous Review）— 全自动化复盘体系，无需人工介入。

```
┌─────────────────────────────────────────────────────────┐
│              🖥️ 多 Agent 协同监控平台                    │
│          http://47.121.129.203:5000                     │
├─────────────────────────────────────────────────────────┤
│  实时采集层                                              │
│  ├─ agent.log 增量解析（~200KB/次，非全量 40MB）         │
│  ├─ SSE 实时推送（事件/日志/Shell 输出）                  │
│  └─ 5 部门状态 + Token 消耗 + 活跃任务                    │
├─────────────────────────────────────────────────────────┤
│  成果提炼层                                              │
│  ├─ 今日成果：技能掌握、能力进化、代码生成统计            │
│  ├─ 里程碑：单日推理突破阈值自动标记                      │
│  └─ 每日大事件：代码战/运维风暴/情报采集叙事化呈现         │
├─────────────────────────────────────────────────────────┤
│  知识沉淀层                                              │
│  ├─ VikingDB 知识库同步 → 每日 08:00（kb-sync.sh）        │
│  ├─ VikingDB 记忆库蒸馏 → 每 6 小时（mem-sync.sh）        │
│  └─ GitHub 仓库自动推送 → push 时触发                     │
└─────────────────────────────────────────────────────────┘
```

### 自动化节奏

| 频率 | 触发方式 | 做什么 |
|:----:|---------|--------|
| 🔄 实时 | SSE 推送 | 监控面板刷新，部门状态更新，事件流水 |
| 🕐 每 10s | 前端轮询 /api/dashboard | 增量解析 agent.log，更新全站数据 |
| 🕗 每日 08:00 | Windows 计划任务 | 知识库同步：检查 GitHub 更新 → 上传 VikingDB |
| 🕐 每 6h | Windows 计划任务 | 记忆库蒸馏：会话经验 E→N→P 三层提炼上传 |
| 🚀 按需 | 用户指令 或 session 结束 | 即时触发记忆蒸馏 / 知识库同步 |

### 经验流转路径

```
遭遇 → 诊断 → 修复 → 记录 → 提炼 → 标准化 → 推送 → 被检索消费
 └──────────── 实时监控 ────────────┘  └── VikingDB 自动化 ──┘
```

**关键设计：** 复盘不是「事后回顾」，而是「实时伴随」。监控仪表盘让你随时看到 Agent 在做什么、花了多少钱、产出了什么。

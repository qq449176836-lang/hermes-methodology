## 二、四层检索优先

在动手做任何事之前，按优先级依次检索：

```
┌──────────────────────────────────────────────────────────┐
│ L0: VikingDB 记忆库（mem-search）                        │
│  │  命中 → 「错题本」历史经验直接复用，跳过推理           │
│  │  未命中 ↓                                             │
├──────────────────────────────────────────────────────────┤
│ L1: VikingDB 知识库（kb-search）                         │
│  │  命中 → 语义检索方法论文档，融合回答                   │
│  │  未命中 ↓                                             │
├──────────────────────────────────────────────────────────┤
│ L2: 精确匹配（会话记忆 / 已知事实）                       │
│  │  命中 → 直接复用，跳过推理                             │
│  │  未命中 ↓                                             │
├──────────────────────────────────────────────────────────┤
│ L3: LLM 语义扩展                                         │
│  │  命中 → 推理适配后使用                                 │
│  │  未命中 ↓                                             │
├──────────────────────────────────────────────────────────┤
│ L4: 遍历 Skill 库                                        │
│  │  命中 → 加载对应技能执行                               │
│  │  未命中 ↓                                             │
│  └── 开始从零构建 → 完成后必须走 ENP 闭环沉淀             │
└──────────────────────────────────────────────────────────┘
```

**核心原则：** LLM 的「推理」成本高、容易幻觉，能直接匹配就绝不推理。

**L0/L1 优先：** VikingDB 记忆库（mem-114f8698）存历史踩坑经验，知识库（kb-4821f4b0271aefe4）存方法论文档和笔记。先搜这两层可以跨会话复用经验，避免重复犯错。

**触发规则：** 涉及运营/数据/策略/技术/经验的问题，优先走 L0/L1 搜记忆库+知识库。寒暄/简单查询跳过。

**例外规则：** 涉及安全/资金/用户数据的问题，即使前几层命中，也必须经过 L3 验证后再执行。

**成本意识：** 不空搜索。只在有明确问题需要解答时才触发 VikingDB 检索，日费用控制在 ¥0.01 以内。

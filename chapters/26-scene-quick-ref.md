## 二十六、场景速查表

| 场景 | 该怎么做 | 不应怎么做 |
|---|---|---|
| 用户给一个新需求 | 先确认理解→三层检索→选执行模式 | 直接开始写代码 |
| 工具调用失败 | 重试1次→查依赖→报告用户 | 反复重试不报 |
| 数据不一致 | 对比源数据→锁定差异→修复→验证 | 直接覆盖 |
| 用户纠正错误 | 确认收到→核实→修正或解释 | 辩解/无视 |
| 重复出现相同错误 | 升级到 P 层（标准化方案） | 每次都当新问题处理 |
| 长时间任务 | background + 进度通知 | 阻塞对话 |
| 需要用户配合 | 给复制粘贴式的步骤 | 给概念性描述 |
| 涉及资金操作 | 等"干"指令 | 自主决策执行 |
| 系统故障 | 自动排查修复 | 等用户发现 |
| 不确定的事 | 说不确定 + 给选项 | 瞎猜或假装知道 |
